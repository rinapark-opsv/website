# Hero v2 기획 — 3개의 AI 팀원, "뿅" 매직 모먼트

> 상태: 기획 디벨롭 중 (2026-06-10). v1 = git 브랜치 `v1`(박제), v2 = `main`(라이브).
> 맥락: v1을 실제 시나리오로 채워보니 **정보 위계가 플랫**하고 **리서처만 이해**되는 플로우. Sun 피드백 = 아이캐칭 필요, '나'를 정의하고 '내가 뭘 할 수 있는지' 보여주기.

> ⚠️ **불변 제약 (2026-06-10 확정)**: **디자인 시스템·제품 목업·챗·아티펙트는 v1과 100% 동일하게 유지한다.** v2에서 바꾸는 것은 **오직 시나리오 재구성**과 그 위에 **v1 디자인 언어로 얹는 3탭**뿐. 새 레이아웃/새 컴포넌트/새 톤 = 금지. "뿅" 연출도 v1의 기존 아티팩트(설문 패널·차트·페르소나 등)와 컴포넌트 안에서 구현한다.

## 1. North Star

> **"한 문장이면, AI 리서치 팀이 결과물을 뿅 — 당신은 거의 아무것도 안 했다."**

- 핵심 메커니즘 = **입력의 사소함 ↔ 결과물의 정교함** 사이의 *간극*.
- 그 간극이 크고, 결과물이 *스스로 조립되는 모습*을 보일수록 → "뿅"(놀라움) + "쉽다"(가능성)가 동시에 증명된다.
- 내러티브: **"문장 하나로 리서치 팀을 고용한다."** 상단 3탭 = 세 명의 AI 팀원.

## 2. 3탭 = 3개의 AI 팀원 = 3개의 매직 모먼트

| 탭 (역할) | 한 줄 약속 | 시그니처 "뿅" | 연출 (조립 안무) |
|:--|:--|:--|:--|
| **AI 리서처** | 한 줄이면 **로직까지 살아있는** 설문 | 문항이 쏟아지고 **분기·파이핑 선이 살아서 그려짐** | 문항 cascade → `if Q1→skip Q5` 점선 화살표 glow draw → `33문항·분기 4·파이핑 6` 카운터 틱업 |
| **AI 분석가** | 데이터 한 번 던지면 **시각화 보고서까지** | 파일 드롭 → **보고서가 차오름**(막대 0→값 sweep) | drop → 제목 타이핑 → 막대그래프 솟음(시퀀셜 블루) → 인사이트 칩 pop |
| **AI 소비자** | 데이터에서 소비자를 **불러내 직접 대화** | 가상 소비자가 **생성되어 말을 걸어옴** | 세그먼트 분리 → 페르소나 카드 *현상*(아바타+속성 채워짐) → 클릭 → 그 사람이 타이핑하며 답장 |

## 3. 시나리오 → 탭 매핑 (현재 8개 기준)

> 원리: **탭당 뿅 1개**. 영웅은 그 뿅을 보여주는 1컷, 심화는 "그게 내 상황에서도 된다"는 근거.

| 탭 (단 하나의 뿅) | 영웅(히어로 1컷) | 심화(Section 1) |
|:--|:--|:--|
| **AI 리서처** · 설문 로직 | **C** 미국 진출 컨셉 검증 설문 | design (설문 → **수집 대상까지**) |
| **AI 분석가** · 보고서 시각화 | **proof** 가설 → 출처 붙은 시각화 보고서 | **입력 다양성**: 구글폼(A) · 내 결과(analyze) · 마켓플레이스(explore) · 팀 이어받기(B) — *어떤 데이터든 같은 엔진* |
| **AI 소비자** · 페르소나 대화 | **persona** 가상 소비자 생성 → 대화 | (비중↑ — 페르소나 다양화 여지) |

## 4. 재구성 표 (유저의 궁금증 → 간소화 아웃풋)

> 원칙: 리서처 용어(크로스탭·유의차·파이핑·웨이브) 제거. 비전문가도 꽂히는 **1인칭 질문**으로 열고, **단일 페이오프 1컷**으로 답한다.

### AI 리서처
| 유저의 궁금증 | 시나리오 Chat (간소화) | 꼭 보여줄 Scene |
|:--|:--|:--|
| "이 컨셉, 미국에서 통할지 **물어볼 설문 빨리 만들 수 있나?**" (C) | "설문 만들어줘" → AI가 목적부터 정리 → 생성 | **33문항 + 로직 선**이 살아서 완성 |
| "리서치 전문 아니어도 **제대로 된 조사를 직접?**" (design) | 설계 제안 → 자동 생성 → 수집 대상 | 설문 생성 + **합성 패널 추천**(수집까지 원스톱) |

### AI 분석가
| 유저의 궁금증 | 시나리오 Chat (간소화) | 꼭 보여줄 Scene |
|:--|:--|:--|
| "**내가 세운 가설, 투자자한테 보여줄 근거가 있을까?**" (proof) | "토너패드 뜬다는 근거 있어?" → Gap 근거 → 보고서 | 한 줄 가설 → **출처 붙은 시각화 보고서** |
| "구글폼 응답, **엑셀 안 건드리고 바로 분석되나?**" (A) | 구글폼 연동 → "요약해줘" → 인사이트·차트 | **자연어 한 마디 → 차트 자동 생성** |
| "신제품 15개 중 **뭐가 제일 잘 팔릴까?**" (analyze) | 결과 연동 → "TOP2 비교" → 순위+연령 | 순위표 + **"30대가 유독 높다"** 한 줄 |
| "우리 데이터 없는데 **이 시장 요즘 뜨나?**" (explore) | "건기식 뭐 떠?" → 마켓플레이스 데이터 | **마켓플레이스에서 끌어온** 트렌드 증감 |
| "내가 한 분석, **팀이 이어서 더 파볼 수 있나?**" (B) | 비교 → 팀 공유 → 팀원이 이어 질문 | **팀원이 같은 채팅에 이어 질문** |

### AI 소비자
| 유저의 궁금증 | 시나리오 Chat (간소화) | 꼭 보여줄 Scene |
|:--|:--|:--|
| "타깃이 누군지도 모르겠는데, **그 사람이랑 직접 얘기해볼 수 있을까?**" (persona) | 타깃 찾아줘 → 두 집단 도출 → 대화 | 가상 소비자와 **실시간 대화 버블** |

## 5. "뿅"을 만드는 4가지 법칙 (Apple / ChatGPT / Claude급)

1. **스왑이 아니라 조립.** 결과물이 그냥 나타나면 안 됨 — 1~1.3초간 스스로 build되는 안무(감속 easing). 빌드 자체가 자랑.
2. **사소한 입력을 항상 노출.** 매직은 *간극*에서 옴. 커서가 한 줄 타이핑 → 엔터 → 기대어 앉는 리듬. 노브 0개.
3. **하나의 영웅 컷에 올인.** 탭당 입력 → 뿅 1컷(8~12초). 나머지는 Section 1.
4. **결과물을 만질 수 있게.** 뿅 직후 0.5초 미세 인터랙션(출처 hover, 페르소나에 한 마디)으로 "진짜네" 박기.

---

## 6. "너무 쉽다"를 보여주는 메타 레이어

- **입력 vs 산출 대비 배지**: 결과물 옆 `입력: 한 문장` · `소요: 10초` 미세 캡션.
- **노브 제로**: 설정·옵션 UI 의도적 은닉. "AI가 목적부터 정리" 장면 = "당신은 안 해도 됨"의 증거.
- **연속된 한 동작**: 탭 전환 = "다음 팀원에게 시키기". 같은 컴포저에 한 줄, 또 뿅.

## 8. 핵심 구조 결정 — 축은 "3개의 뿅" (axis 재정의)

### 왜 분석가에 5개가 몰렸나 (표면 ≠ 진짜)
- 표면: 3탭을 **아웃풋 축**(설문/보고서/대화)으로 잡으니 "분석으로 끝나는" 5개가 한 칸에 쌓임.
- 진짜: **v1 시나리오는 "데이터 상황" 축으로 기획**됐다 — 구글폼 / 내 결과 / 마켓플레이스 / 팀 / 가설. 리서치 업무 대부분이 분석으로 귀결되니, 아웃풋 축에서 세면 한 칸에 모인 것. **중복이 아니라 축 미스매치.**
- 정정된 진단: "개수가 많다"가 아니라 **"5개의 서로 다른 가치 스토리가 하나의 히어로 슬롯을 두고 경쟁해 강조가 흐려진다(=플랫)."** → 강조의 문제.

### 결정: 탭의 조직 원리 = "3개의 시그니처 뿅"
탭은 *시나리오를 분류해 담는 통*이 아니라 **하나의 매직을 한 번 보여주는 무대**다.

| 탭 | 그 탭의 단 하나의 뿅 |
|:--|:--|
| AI 리서처 | **설문 로직** 살아남 |
| AI 분석가 | **보고서 시각화** 차오름 |
| AI 소비자 | **페르소나** 현상→대화 |

→ 분석은 자연히 **1개의 뿅(보고서 시각화)** 으로 수렴. `A·B·analyze·explore`는 경쟁 데모가 아니라 **"입력 다양성" 증거**로 재배치 — **"무슨 데이터를 갖고 있든 똑같이 뿅."**

### 분석가 탭의 "입력 다양성" (Section 1 심화)
같은 *분석→시각화* 엔진에 물리는 입력 매트릭스로 한 줄씩:
- 내 **구글폼**(A) · 내 **컨셉 결과**(analyze) · 데이터 없을 때 **마켓플레이스**(explore) · **팀이 이어받기**(B)
- → 5개 경쟁이 아니라 *"어떤 데이터든 된다"* 한 메시지의 근거.

### 남는 정비
- **AI 소비자**(persona)는 1개지만 가장 신선 → 비중↑, 데모의 시그니처 후보.
- **C↔design 중복** → 히어로엔 C(컨셉 검증), design은 "수집까지" 심화로 흡수.
- **전문용어**(크로스탭·파이핑·유의차·웨이브)는 히어로 제거, 심화에서만.
- (열린 변수) 분석가 사용 비중이 실제 제품에서 제일 크다면, 분석가에 히어로 비트 2개(내 데이터 / 데이터 없을 때)를 주는 안도 재고 가능 → §9.
