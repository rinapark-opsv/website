## 3. 인터랙션 모듈 기본 구동 단계 (공통 씬 풀)

| 씬 ID | 씬 이름 | 좌측 채팅 | 우측 프리뷰 |
| :--- | :--- | :--- | :--- |
| S1 | **데이터셋 선택** | "분석에 참조할 데이터를 선택합니다" | 데이터셋 드롭다운 / 업로드 / 라이브러리 |

| S5 | **변수 재코딩 / 크로스탭** | "한국+일본을 East Asia로 묶어 비교" | 크로스탭 테이블 |

## 시나리오 1 — 리서치 설계 (브리프 → 설문) · `id: C`

**태그라인**: "브리프 한 줄이 검증된 설문 문항으로, 10분 만에"
**구동 씬**: S2 → S3 → S7 → S3

| 씬 | 좌측 채팅 내용 | 우측 프리뷰 내용 |
| :--- | :--- | :--- |
| S2 | "미국 MZ 여성 대상 K뷰티 구매 의향 조사 브리프 써줘" | 프롬프트 입력창 (`prompt`) |
| S3 | Research Brief 생성 — 목적/타겟/핵심 질문 + Source 1 | 브리프 문서 패널 (`brief`) |
| S7 | "다음 단계를 선택하세요" — 설문 문항 설계 / 타겟 패널 추천 / 경쟁 브랜드 분석 | Next Step 선택지 (`nextstep`) |
| S3 | 설문 문항 초안(Q1–Q5) 생성 + 검증된 문항 뱅크 출처 | 문항 리스트 패널 (`survey`) |

## 시나리오 2 — 내 설문 로데이터 분석 · `id: A`

**태그라인**: "로데이터를 올리면, 가공 없이 바로 분석이 시작됩니다"
**구동 씬**: S1 → S2 → S3 → S4 → S6 → S8 *(스펙 A와 동일)*

| 씬 | 좌측 채팅 내용 | 우측 프리뷰 내용 |
| :--- | :--- | :--- |
| S1 | 설문 파일 업로드 완료 — `survey_rawdata_2025.xlsx` (+2,400 rows) | 파일 업로드 + 데이터셋 등록 (`upload`) |
| S2 | "이 설문 데이터의 주요 인사이트를 요약해줘" | 프롬프트 입력창 (`prompt`) |
| S3 | Summary 3개 — 브랜드 인지도/재구매 의향/구매 채널 + Source 1 | 인사이트 패널 (`insights`) |
| S4 | "20대 응답자만 별도로 분석해줘" → 20s key insights | 필터 적용 → 결과 갱신 (`filter`) |
| S6 | "연령대별 결과를 bar chart로 보여줘" → 차트 파일 생성 | 차트 렌더링 (`chart`) |
| S8 | "분석 결과가 프로젝트에 저장되었습니다" | 프로젝트 오버뷰 (`overview`) |

## 시나리오 3 — 경쟁/트렌드 비교 + 팀 협업 · `id: B`

**태그라인**: "분석이 팀의 자산이 되고, 누구나 이어서 질문합니다"
**구동 씬**: S1 → S2 → S3 → S5 → S9 *(스펙 B에서 S7 공유모달은 S9에 통합)*

| 씬 | 좌측 채팅 내용 | 우측 프리뷰 내용 |
| :--- | :--- | :--- |
| S1 | 트렌드 데이터셋 선택 — `2025 AI Search Trend Report (KR+US+JP)` (n=3,000) | 데이터셋 표시 (`dataset`) |
| S2 | "한국과 미국의 소비자 AI 검색 행동 차이를 비교해줘" | 프롬프트 입력창 (`prompt`) |
| S3 | 크로스컨트리 Summary — KR 체험률 / US 만족도 / JP 신중 + Source 1 | 비교 인사이트 패널 (`insights2`) |
| S5 | "한국+일본을 East Asia로 묶어 미국과 크로스탭 해줘" → 유의차 테이블 | 크로스탭 테이블 (`crosstab`) |
| S9 | 팀원 공유 → 지연(팀원) 아바타 "일본만 따로 보면?" → 결과 이어보기 | 팀 협업 채팅 (`collaborate`) |

## 시나리오 4 — 브랜드 인식 조사 설계·생성·수집 · `id: design`

**페르소나**: 김민정 팀장 · 리서치 수행
**태그라인**: "질문 한 줄이 검증된 설문으로, 10분 만에"
**구동 씬**: S2 → S3 → S3* → S7 → S2·S3 → S3* *(제품 풀플로우 — `*`는 제품 고유 씬)*

| 씬 | 좌측 채팅 내용 | 우측 프리뷰 내용 (인터랙션) |
| :--- | :--- | :--- |
| S2·S3 | "우리 브랜드 인식이 어떤지 알고 싶어요" → 접근 방향 제안(인지도·이미지 / 포지셔닝) | 프롬프트 (`prompt`) |
| S3 | "MZ세대 중심으로 설문 만들어줄 수 있어?" → 프로젝트 선택 카드 + 설계 제안 | **프로젝트 선택 → 프로젝트 데이터 화면 자동 오픈** (`projectdata`) |
| S3* | 프로젝트 선택 → 생성 과정 카드 3종(목표 정리 / 개요 정리 / 문항 작성) | 설문 생성 진행 (`designsurvey`) — *제품 고유* |
| S7 | 완료 카드 — 브리프/설계서/문항 완료 + 데이터 로우 "설문이 생성되었어요!" | **더 보기 → 브리프·설계서 문서 자동 오픈** (`designbriefdoc`/`designplandoc`) |
| S2·S3 | "3번 문항에 '가격이 비싸서' 보기를 추가해줘" → 수정 반영 + 데이터 로우 | 설문 미리보기(수정 반영) (`designsurvey`) |
| S3* | "좋아, 바로 수집 시작할게" → 수집 대상 선택(합성패널 추천/소비자 패널/내 고객) | **수집 시작 → 수집 조건(쿼터 표) 자동 오픈** (`collectdoc`) — *제품 고유* |

## 시나리오 5 — 신제품 컨셉 테스트 결과 분석 · `id: analyze` *(디폴트 자동재생)*

**페르소나**: 박준호 PM · 결과 활용
**태그라인**: "교차분석·세그먼트 유의차까지, 리서처처럼 답합니다"
**구동 씬**: S1 → S2 → S3 → S2 → S3(S5형 교차)

| 씬 | 좌측 채팅 내용 | 우측 프리뷰 내용 |
| :--- | :--- | :--- |
| S1 | 컨셉 테스트 결과 연동 — `2026 K뷰티 신제품 컨셉 테스트` (15개 컨셉, n=1,164) | 데이터셋 연동 (`conceptds`) |
| S2 | "15개 컨셉의 구매 의향을 TOP2 기준으로 비교해줘" | 프롬프트 (`prompt`) |
| S3 | 컨셉 TOP2 비교 — 상위 VT/AESTURA/Primera, 인라인 표 + Source 1 | 컨셉 비교 표 (`concepts`) |
| S2 | "TOP2 60% 이상 제품 3개를 연령대별 구매의향으로 비교해줘" | 프롬프트 (`prompt`) |
| S3 (S5형) | 연령대별 교차 — 30–39세 최고(약 70%), 신뢰수준 80% 유의차 색상 | 세그먼트 교차 표 (`agecross`) |

**페르소나**: 이수민 마케터 · 근거 필요
**태그라인**: "보유 데이터가 없어도, 라이브러리로 근거를 만듭니다"
**구동 씬**: S1 → S2 → S3(S5/S6형 웨이브 비교) → S2 → S3

| 씬 | 좌측 채팅 내용 | 우측 프리뷰 내용 |
| :--- | :--- | :--- |
| S1 | 자체 데이터 없음 → 라이브러리 탐색 — `건강기능식품 트렌드 데이터 (2025·2026)` (출처 2건) | 라이브러리 데이터 (`library`) |
| S2 | "건강기능식품에서 2025 대비 2026에 뜨는 카테고리 알려줘" | 프롬프트 (`prompt`) |
| S3 (트래킹) | 웨이브 비교 — 오메가3 +2.7%p 등 증가, 인라인 표 + 출처 1·2 | 웨이브 비교 표 (`wave`) |
| S2 | "30대 타깃으로 더 깊이 분석해줘" | 프롬프트 (`prompt`) |
| S3 | 30대 핵심 인사이트 — 속성/제형/채널/가격, 출처 1 | 세그먼트 인사이트 (`trendins`) |
